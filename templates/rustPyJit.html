What is a Just in Time Compiler?

A Just in Time Compiler is Part of a Larger Interpreter, an Intepreter typically functions by compiling all code to some tokenized output. This is similar to compiled languages like C and Java. However where Interprters differ from compilers is on the backend, where this tokenized output is then processed line by line instead of all at once. This is handled by the VM in both the CPython and RustPython Interpreters. However This Process of Intepreting Code via the VM is often very costly and intorodcues unecesarry overhead as we often spend time re-intepreting previosuly processed instructions. Just-In-Time Compilers aim to eliminate this overhead by Compiling said frequently excuted Code into machine code eliminating the overhead. Some More Flushed out JIT Compilers work dynamically like the CPython Intepreter post 3.13, however the current RustPython Jit works on demand via the .__jit__() flag.
